# PROJECT_ANALYSIS.md

## 프로젝트 개요

**한 줄 요약**: 졸업생 기금 납부 내역을 분석하여 미납/부족/초과 오류를 자동 검출하는 Streamlit 웹 애플리케이션

**타겟 사용자**: 기금 관리 담당자, 행정 실무자

**핵심 가치**: 수동 엑셀 검토 작업을 자동화하여 오류 검출 시간 단축, 졸업생 명단 기반 필터링으로 업무 효율성 증대

---

## 아키텍처 결정 사항

### 왜 이 스택을 선택했는가

| 결정 | 이유 | 트레이드오프 |
|------|------|-------------|
| **Streamlit** | 데이터 분석 앱을 빠르게 구축 가능, Python 생태계와 자연스러운 통합 | 커스텀 UI 제약, 대규모 서비스에는 부적합 |
| **DB 없이 메모리 처리** | 설치/운영 복잡도 최소화, 엑셀 기반 워크플로우와 자연스러운 연결 | 대용량 데이터 처리 제약, 세션 간 상태 유지 불가 |
| **Google Sheets 공개 URL** | API 인증 없이 간단히 연동 가능 | 비공개 시트 접근 불가, 할당량 제한 |
| **Pandas DataFrame** | 엑셀 데이터 처리에 최적화, 벡터화 연산으로 성능 확보 | 메모리 사용량 증가 |

### 데이터 흐름 원칙

1. **상태 관리**: 모든 상태는 Streamlit 세션 상태(st.session_state)로 관리. 페이지 새로고침 시 상태 초기화됨.
2. **데이터 캐싱**: 엑셀 생성 결과는 `@st.cache_data`로 캐싱하여 재연산 방지.
3. **처리 순서**: 입력 검증 → 졸업생 필터링 → 최초납부월 추출 → 오류 검출 → 결과 병합

## 핵심 비즈니스 로직

### 오류 검출 규칙

| 기금 | 기준 | 오류 판정 |
|------|------|----------|
| 운영기금 | 코드 11~17 중 하나라도 납부해야 함 | 미납 (모두 없음) |
| 협력기금 | 운영기금 총액 × 30% (2019.03 이전) / 40% (이후) | 부족/초과 (1,000원 이상 차이) |
| 복지기금 | 운영기금 총액 × 100% | 부족/초과 (1,000원 이상 차이) |

### 최초 납부월 필터링

- **목적**: 졸업 전 납부 기록을 오류 검출에서 제외
- **판정 조건**: (코드1=1 & 코드2=1~7) 또는 (코드1=2,3 & 코드2=1)
- **동작**: 최초 납부월 이전의 미납 데이터는 결과에서 제외, 제외 건수는 UI에 표시

### 기간별 비율 변경 (협력기금)

- **2019년 3월까지**: 운영기금의 30%
- **2019년 4월부터**: 운영기금의 40%
- **이유**: 기금 규정 개정 반영

## 데이터 관계 요약

### 입력 데이터 관계

```
원본 엑셀 (납부내역)
    │
    ├── 이름 ──────────┐
    │                  │
    └── (이름,년,월)   │  → 그룹핑 키
                       │
Google Sheets (졸업생) │
    │                  │
    └── 이름 ──────────┘  → 조인으로 졸업생만 필터링
```

### 출력 데이터 구조

- **오류검출결과**: (이름, 납부년, 납부월, 기금명)별 오류 상세
- **오류요약**: 이름별 오류 건수 집계
- **최초납부월**: 이름별 최초 납부 시작월

## 개발 시 주의사항

### 자주 발생하는 실수

1. **컬럼명 하드코딩**: `constants.py`의 `Col` 클래스를 사용하지 않고 문자열 직접 사용 → 컬럼명 변경 시 오류
2. **상태값 불일치**: `Status` 클래스와 `app.py`의 `counts` dict가 일치하지 않으면 UI 카운터 오류
3. **년월 타입**: `해당년`, `해당월`은 int로 변환 필요. 문자열로 들어오면 그룹핑 실패
4. **최초납부월 조건**: 코드 조건을 잘못 설정하면 졸업 전 데이터가 포함되거나 제외됨

### 성능 고려사항

- **UI 렌더링**: 상위 200건만 표시. 전체 데이터는 엑셀 다운로드로 확인.
- **엑셀 생성**: `to_excel_bytes()` 호출은 무거운 연산. 캐싱 필수.
- **Google Sheets**: 네트워크 지연 가능. 로딩 상태 표시 필요.

### 보안 체크포인트

- [ ] Google Sheets URL은 공개만 지원 (비공개 시트 접근 불가)
- [ ] 업로드 파일 최대 20MB (Streamlit 기본 설정)
- [ ] 민감 데이터 로그 금지 (이름, 금액 등)
- [ ] 출력 엑셀에 민감 정보 포함 여부 확인 필요
